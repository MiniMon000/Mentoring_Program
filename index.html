1. Buff & Loot Actions

Priority
Training Progress 0%

Concepts

  • Loot lists structure and inheritance
  • Buff Tuning relationships
  • Additional/Removal Mechanics
  • Conditional logic for loot triggers

Implementation Plan

Extract Tuning "CheatingIntensityBuff_xxx".

Have a review with all parameters.

Extract Tuning "Debug_Loot_Buff_DebugIntensity_xxx".

Have a review "buffs_to_remove" and "buff".

Extern Tuning "phone_BrowseWebsites" which is a Game Base interaction.

Add outcome for result of Interaction.

Add Tuning ID to Snippet.

Add XML Injector ts4script into the Mods Folder.

<L n="buffs_to_remove">
<T>12345<!--Buff: Buff_Sad--></T>
</L>
<U n="buff">
<V n="buff_reason" t="enabled">
    <T n="enabled">0xDE43ECF7<!--(From SnootySims)--></T>
</V>
<T n="buff_type">54321<!--Buff: Buff_Happy--></T>
</U>

Example Assignment

"Browse Web" in "Sad", be "Happy";"Browse Web" in "Happy", be "Sad":

  • "Loot Action" for Mood "Happy"
  • "Loot Action" for Mood "Sad"
  • Duration "Browse Web" in Phone, check the Mood of Sim, if "Sad", be "Happy" and vice versa!

2. Custom Interactions

Intermediate
Training Progress 0%

Concepts

  • Interaction from extract Tuning
  • Outcome functionals
  • Animation mapping
  • Perform functional on Phone
  • Perform functional on Computer

Implementation Steps

  1. Clone existing interaction from Base Game.
  2. Modify display text at "display_name".
  3. Modify animation at "factory".
  4. Add new action from Requirement 1 to Phone at "outcome".
  5. Add new action from Requirement 1 to Computer at "outcome".
  6. Add Tuning ID to Snippet.
  7. Add XML Injector ts4script into the Mods Folder.

Example Assignment

"Browse Web" in "Sad", be "Happy";"Browse Web" in "Happy", be "Sad":

      On Phone:
  • Animation with "Browse Web" changed to "Playing Game"
  • "Loot Action" for Mood "Happy"
  • "Loot Action" for Mood "Sad"
  • Duration "Browse Web" in Phone, check the Mood of Sim, if "Sad", be "Happy" and vice versa!
      • On Computer:
  • Animation with "Browse Web" changed to "Playing Game"
  • "Loot Action" for Mood "Happy"
  • "Loot Action" for Mood "Sad"
  • Duration "Browse Web" in Computer, check the Mood of Sim, if "Sad", be "Happy" and vice versa!

3. Clone Objects Modification

Advanced
Training Progress 0%

Key Components

  • Object Definition (.OBJD) cloning
  • Tuning Resource (.TUNING) inheritance
  • Rigs/Slot additional
  • Custom affordances
  • Perform Object functionals

Implementation Strategy

Step 1: Use S4S to clone base object
Step 2: Modify object's Name, Price, Description
Step 3: Modify object and interactions in tuning
Step 4: Add new interactions at "_super_affordances"
Step 5: Perform Object

Example Assignment

Create a "Camera" object clone that:

  • Change name to "Camera Selfie"
  • Add new functional for Selfie available to Sim only Camera in Sim's inventory
